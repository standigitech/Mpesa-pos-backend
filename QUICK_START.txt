🔑 CONFIGURE CREDENTIALS
═════════════════════════════════════════════════════════════════════════════

Edit: mpesa_backend/.env

CONSUMER_KEY=           ← From Safaricom
CONSUMER_SECRET=        ← From Safaricom
SHORTCODE=              ← Your till number
PASSKEY=                ← M-Pesa generated key
CALLBACK_URL=           ← Your domain/ngrok URL
NODE_ENV=development    ← or staging/production

✅ API ENDPOINTS
═════════════════════════════════════════════════════════════════════════════

Health Check:
  GET http://localhost:3000/api/health
  Response: {status: "ok", timestamp, uptime}

Root:
  GET http://localhost:3000/
  Response: {message, environment, version}

STK Push:
  POST http://localhost:3000/api/stkpush
  Body: {"phone": "254712345678", "amount": 1000}
  Response: {MerchantRequestID, CheckoutRequestID, ResponseCode}

Callback:
  POST http://localhost:3000/api/callback
  (Receives M-Pesa responses)

🐛 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

Port already in use?
  → Change PORT in .env

M-Pesa auth fails?
  → Check CONSUMER_KEY and CONSUMER_SECRET

Phone validation error?
  → Use format: 254712345678 (no + or 0 prefix)

Callback not received?
  → Ensure CALLBACK_URL is publicly accessible HTTPS

Server won't start?
  → npm install (install dependencies)
  → Check .env file exists
